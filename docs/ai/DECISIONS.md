# Архитектурные решения

## ADR-001: клиентский односервисный frontend

- Статус: Accepted
- Дата: 2026-06-28
- Решение: сохранять проект как один React/Vite frontend-сервис без добавления backend-компонентов до появления соответствующей продуктовой задачи.
- Причина: это соответствует заявленной границе проекта и фактической структуре репозитория.

## ADR-002: постоянная память в `docs/ai/`

- Статус: Accepted
- Дата: 2026-06-28
- Решение: считать файлы `docs/ai/` единственным источником постоянного контекста проекта и обновлять их вместе с изменениями кода.
- Причина: проектный контекст должен быть проверяемым, версионируемым и независимым от истории чата.

## ADR-003: текущий контактный канал зафиксирован как baseline

- Статус: Superseded by ADR-004
- Дата: 2026-06-28
- Решение: до отдельной задачи сохранять существующее поведение контактов: `mailto:` и опциональная внешняя Telegram-ссылка, без серверной отправки.
- Причина: это фактическое рабочее поведение репозитория; причина исходного выбора в коде и истории проекта не документирована. Решение не определяет целевую реализацию будущей доработки.

## ADR-004: Telegram-доставка через серверную точку frontend-сервиса

- Статус: Superseded by ADR-007
- Дата: 2026-06-28
- Решение: обрабатывать `POST /api/contact` в том же Node-процессе, который обслуживает frontend, и вызывать Telegram Bot API только на сервере.
- Причина: токен бота нельзя безопасно помещать в Vite-клиент; единый Node-процесс сохраняет ограничение проекта в один сервис без выделения отдельного backend.

## ADR-005: два Telegram-сообщения при наличии файла

- Статус: Accepted
- Дата: 2026-06-28
- Решение: всегда отправлять посреднику полную текстовую заявку, затем при наличии файла отправлять второй запрос с документом без caption.
- Причина: подпись Telegram-документа ограничена по длине, поэтому она не гарантирует передачу всех полей длинной заявки.

## ADR-006: production запускается собственным Node-сервером

- Статус: Accepted
- Дата: 2026-06-28
- Решение: в Coolify запускать `npm start`; сервер по умолчанию слушает `0.0.0.0:4173`.
- Причина: `vite preview` раздаёт только frontend-статику и не обслуживает `POST /api/contact`, а reverse proxy Coolify должен иметь доступ к серверу за пределами loopback контейнера.

## ADR-007: Telegram доступен только через noteapp-ai-integration

- Статус: Accepted
- Дата: 2026-06-29
- Решение: Node-сервер лендинга вызывает защищённый `POST /api/social/posts` сервиса `noteapp-ai-integration`; прямые запросы к `api.telegram.org` удалены.
- Причина: Telegram Bot API недоступен из локации deployment лендинга, а существующий посредник уже реализует `X-API-Key`, Telegram text/document и безопасную обработку credentials.
